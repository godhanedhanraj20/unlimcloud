# Unlim Cloud Development Roadmap

This document outlines the development roadmap for the Unlim Cloud web application, based on the vision for a modern, user-friendly cloud storage experience.

## Phase 1: Core Experience & Information Hub [Skipped]

This phase, which focused on building out the informational pages, has been skipped to prioritize feature development.

## Phase 2: "Google Photos" Style Gallery [In Progress]

This phase focuses on creating a modern, intuitive, and visually appealing gallery for managing photos and videos.

-   **[To Do]** **UI/UX Design:**
    -   Design a grid-based layout for photo and video thumbnails, similar to Google Photos.
    -   Implement responsive design to ensure a great experience on all screen sizes.
    -   Create designs for viewing individual items, including metadata display (name, date, size).
-   **[To Do]** **Drag-and-Drop Upload:**
    -   Develop a file upload component that supports dragging and dropping files directly onto the gallery interface.
    -   Provide visual feedback during the upload process (e.g., progress bars).
-   **[To Do]** **Backend Integration:**
    -   Develop API endpoints for uploading, fetching, and deleting gallery items.
    -   Integrate the front-end gallery with these APIs.
-   **[To Do]** **Features:**
    -   Implement image and video thumbnail generation for fast previews.
    -   Add functionality to view media in a full-screen lightbox.
    -   (Optional) Add features like creating albums or sorting by date.

## Phase 3: "Google Drive" Style File Management

This phase focuses on building a powerful and familiar file management interface, enabling users to organize all their files, not just media.

-   **[To Do]** **UI/UX Design:**
    -   Design a file explorer interface with options for both list and grid views.
    -   Incorporate familiar elements like a navigation bar (breadcrumbs), a sidebar for folders, and a main content area for files.
    -   Design modals and context menus for actions like 'Rename', 'Move', 'Delete', and 'Get Info'.
-   **[To Do]** **Core Functionality:**
    -   Implement file and folder creation, renaming, and deletion.
    -   Develop the logic for moving files and folders between different locations.
    -   Enable file uploads (via button and drag-drop into folders) and downloads.
-   **[To Do]** **Backend Integration:**
    -   Develop the necessary API endpoints to support all file and folder management operations.
    -   Connect the UI to the backend to reflect the file system state accurately.
